Comments

1. Overall Structure

This website consists of four main pages.

  • index.html - the homepage
  • past.html - describes my past experience with computing
  • future.html - describes my future goals
  • comments.html - explains the design of the website

All pages are connected using a navigation menu. Users can easily move between pages using links at the top of each page.

2. Internal Structure of Each Page

Each page follows a consistent layout:

  • A navigation menu at the top
  • A main heading (h1)
  • Content organised into paragraphs (p)
  • Images with alt descriptions

Div elements are used to group content, making the structure clear and easy to read.

3. Styles Used

Selector Type Purpose Used In
body Element Sets the font and applies a light background colour All pages
.menu Class Styles the navigation bar with background colour and spacing All pages
.menu a Class selector Styles navigation links with colour, size and spacing Navigation menu
.menu a:hover Pseudo-class Changes link colour when hovered to improve interactivity Navigation menu
.section Class Uses flexbox to arrange content and images side by side Past / Future pages
.reverse Class Reverses the layout direction of flex items Content sections
.text Class Controls width, padding and background for text areas Content sections
.image img Class Sets image size and rounded corners All images
#footer ID Styles the footer with background colour and alignment Bottom of pages
.grid Class Uses grid layout to organise content into columns Future / gallery section

4. Graphic Design

  • A simple layout is used to improve readability.
  • The light blue background will make people look more comfortable.
  • Images are included to make the website more engaging.
  • Fonts are easy to read and consistent across pages.

5. Accessibility

Use of alt tags:

  • All images include alt attributes.
  • This helps users understand the content if images do not load.
  • It also improves accessibility for screen readers.

CSS turned off:

  • The website remains readable without CSS.
  • Content is still organised using headings and paragraphs.
  • Users can still navigate using links.
